Mission

HJB provides access to JMS resources via HTTP

Scope

HJB

  • will allow any language/library capable of communication via HTTP to access JMS-compliant messaging products
  • aims to provide a RESTful equivalent to all of the non-optional portions of the JMS API
  • JMS access include sending/receiving of all types of JMS message, registration of JNDI configured provider resources and connection/session management

    • Implementation overview

    HJB
    • is deployed as a servlet (HJBServlet) that can run on any compliant Servlet container
    • uses standard Java API libraries and APIs, so as to remain JMS vendor agnostic
    • does not concern itself with session management, authentication and security - it acts an HTTP origin server for JMS resources, these other features can be added by other HTTP servers in the request processing chain. These could include the servlet container in which the HJB servlet resides, or any other server with the necessary features e.g., Apache
    • is extensively unit tested using JUnit and JMock
    • is fully internationalized
