HJB (HTTP JMS Bridge) links the HTTP protocol to the JMS API.

HJB 

* Provides a 'RESTful' API equivalent to the JMS API, i.e, it becomes
possible to perform JMS operations by accessing JMS objects as
resources via HTTP

* Covers most of the non-optional parts of the JMS API

* Provides encoding/decoding of all JMS message types to a textual
form suitable for transmission as text in a HTTP message, including the
JMS header and property values
